# Proyecto de Simulación

## Ecosistema de Aseguramiento de la Calidad del Software

## 🎯 Objetivo General

## Diseñar y ejecutar un Plan de Aseguramiento de la Calidad del Software (PACS), centrado en la aplicación

## de técnicas estáticas a requerimientos, arquitectura y programas, con apoyo de un ecosistema tecnológico

## integrado que permita soportar, automatizar y documentar el proceso conforme a estándares

## internacionales. En la primera entrega solo la aplicación de técnicas estáticas a requerimientos y

## arquitectura y en la segunda estáticas a código y dinámicas a partir de pruebas.

## 🧩 Descripción del Proyecto

## Cada equipo deberá construir un ecosistema de tecnologías que les permita:

1. Diseñar el Plan de Aseguramiento de la Calidad del Software (PACS).
2. Ejecutar la primera fase del PACS, enfocada en la aplicación de técnicas estáticas sobre requerimientos y
    arquitectura.
3. Aplicar listas de chequeo para evaluar la calidad de los requerimientos y la arquitectura, con base en
    estándares como IEEE e ISO/IEC/IEEE, a partir del refinamiento de una lista de chequeo de una muestra
    entregada por el docente en las presentaciones de Unidad II.
4. Documentar el proceso completo: roles, herramientas, métricas y resultados.

## ⚠️ Importante: El PACS debe estar alineado con estándares internacionales como:

```
 ISO/IEC 25010 : Modelo de calidad del producto software.
 ISO/IEC 12207 : Procesos del ciclo de vida del software.
 ISO/IEC/IEEE 29148 : Requisitos de sistemas y software.
 IEEE 730 : Estándar para planes de aseguramiento de la calidad del software.
 ISO/IEC/IEEE 42010 : Descripción de arquitectura
```

## 📦 Entregables

## 1. Infograma del Ecosistema Tecnológico

```
o Visualización clara de las herramientas seleccionadas y su integración.
o Debe incluir esquemas de integración entre herramientas (por ejemplo, cómo se
conectan Jira con Confluence, GitHub, gemini).
o Categorías mínimas: Aplicación de estática y dinámica, métricas, automatización,
reportes, gestión documental, IA’s para el apoyo del proceso.
```
## 2. Invitación al Espacio de Trabajo Compartido

```
o Acceso de visualización para el docente.
o Debe permitir observar la administración de documentos, históricos de cambios y
trazabilidad por roles y sus actividades, así como la relación entre los artefactos
generados en el contexto de SQA.
```
## 3. Lista de Chequeo Refinada

```
o Basada en la muestra entregada.
o Justificada según estándares de calidad de requerimientos y arquitectura
```
## 4. Plan de Aseguramiento de la Calidad del Software (PACS)

```
o Documento estructurado con objetivos, alcance, técnicas, métricas, herramientas y
cronograma.
```
## 5. Evidencia de Ejecución de la Fase Estática

```
o Aplicación de la lista de chequeo sobre un conjunto de requerimientos y la arquitectura.
o Resultados documentados y analizados. Debe haber un plan de revisiones y un informe
de revisiones como parte de los documentos vinculados con el plan de aseguramiento.
```
## 6. Reflexión sobre el Ecosistema Tecnológico


```
o Breve análisis crítico sobre las decisiones tomadas en la selección e integración de
herramientas. ¿Cuáles herramientas de IA apoyan el proceso y cómo?
o Debe considerar ventajas, limitaciones y oportunidades de mejora.
o Tanto el infograma como la reflexión deben ser apéndices del plan de aseguramiento.
```
